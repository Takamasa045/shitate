# Shitate（シタテ）

**AIキャラクターの「設定」「見た目」「育てた履歴」を、ひとつの場所に残すためのローカルツールです。**

画像生成AIで気に入ったキャラクターができても、次に作ると顔や衣装が変わってしまう。
どのプロンプトが良かったのか、あとから分からなくなる。

Shitate は、そんな試行錯誤をキャラクターごとに整理します。
設定・プロンプト・お手本画像・改善メモを残せるので、次の日も、別の画像生成ツールでも、同じキャラクターの続きを作れます。

> Shitate 自体は画像を生成しません。キャラクターを設計し、生成に使うプロンプトを整え、結果を記録する場所です。
> 配布版にはキャラクター画像を含めず、画像なしのオリジナルサンプルだけを収録しています。

## まずはCodex / Claude Codeで始める

Shitateは、ターミナルのコマンドを自分で覚えなくても使えます。
Forkは「自分のGitHub上にコピーを作ること」、cloneは「PCで使える作業用コピーを作ること」です。

1. [このリポジトリ](https://github.com/Takamasa045/shitate)をcloneします
2. 自分のGitHub上にコピーを持ちたい場合は、先に **Fork** してから、そのForkをcloneします
3. cloneした `shitate` フォルダを [Codex](https://openai.com/codex/get-started/) または [Claude Code](https://code.claude.com/docs/en/desktop-quickstart) で開きます
4. 次の文章を、そのままチャットへ貼り付けます

```text
このShitateリポジトリを使える状態にセットアップしてください。
AGENTS.mdとREADME.mdを読み、必要な環境を確認して、不足があれば分かりやすく案内してください。
準備ができたら、同梱のランチャーでShitate Studioを起動し、サンプル「和紙狐」で基本操作を案内してください。
その後、私の最初のキャラクター作成を質問しながら進めてください。
画像は、私が明示するまで生成しないでください。
最後に、次に私がCodex / Claude Codeへ頼める一言を示してください。
```

あとはCodex / Claude Codeが、必要な準備の確認、Studioの起動、最初の操作案内まで進めます。
途中で実行許可を求められた場合は、内容を確認して許可してください。

詳しい流れは [Codex / Claude Codeでの始め方](QUICKSTART.md) にまとめています。

## こんなときに使えます

- 同じキャラクターを、何度も安定して登場させたい
- 顔・衣装・色・持ち物などの「変えてはいけない部分」を残したい
- 何を直したら良くなったのか、試行錯誤を次回へ引き継ぎたい
- 画像だけでなく、動画制作でも同じキャラクターを使いたい
- 歴史上の人物や物語の登場人物を、資料に沿って視覚化したい

## Shitate に残せるもの

キャラクターごとに、次の内容がひとまとまりになります。

- **人物設定** — 名前、性格、役割、外見の特徴
- **世界観** — いつ、どこで、どんな物語を生きているか
- **基本の見た目** — 毎回守りたい顔、体格、衣装、色
- **場面ごとの差分** — 三面図、表情、ポーズ、特定のシーン
- **お手本画像** — 同じ顔や衣装を再現するための anchor
- **改善日記** — 試したこと、評価、次に直すこと

ファイルは一般的な Markdown・YAML・JSON が中心です。
特定のAIサービスに閉じ込められず、将来ツールが変わっても持ち運べます。

## Studioの起動も自然な言葉で頼めます

Shitate Studio は、キャラクターを見たり育てたりするためのブラウザ画面です。
Codex / Claude CodeでShitateのフォルダを開いたまま、次のように頼んでください。

> Shitate Studioを同梱のランチャーで起動して。ブラウザが開いたら、今の画面で最初にすることを案内して。

初回だけ、動作に必要なファイルを自動で準備します。準備が終わるとブラウザが開きます。
2回目以降も「Studioを起動して」と頼めます。

Studioを終了するときも「Shitate Studioを終了して」と頼めます。自分で終了する場合は、起動時に開いたターミナルで `Control + C` を押します。
キャラクターのデータはこのフォルダ内に保存され、Studio が勝手に外部へアップロードすることはありません。

AIを使わず自分で起動したい場合だけ、Macでは `Shitate Studio.command`、Windows / Linuxでは `node scripts/launch-studio.mjs` を使います。

## 画面の言葉

Studioでは、制作の流れに合わせて次の名前を使っています。

- **人物録** — キャラクターの一覧。新しいキャラクターもここから作ります
- **概観** — 名前、役割、現在の状態などのまとめ
- **詞書（ことばがき）** — キャラクターの基本設定と、場面ごとのプロンプト
- **調合** — 基本設定と場面指定を、画像生成用の文章にまとめる操作
- **巻物** — これまでに保存したプロンプトの一覧
- **手本** — 採用したお手本画像。画面では anchor とも表記します
- **日録** — 試した結果、評価、次に直すことを残す制作メモ

## 基本の使い方：自然な言葉で一巡させる

画面を自分で操作しても、Codex / Claude Codeに頼んでも構いません。
Markdownを直接編集する必要はありません。

### 1. キャラクターを作る

「人物録」→「新しいキャラクター」を開き、IDと表示名を入力します。役割は任意です。

> 新しいキャラクターを作りたい。名前や役割、見た目について、必要なことを一つずつ質問して。

- **ID** — 英小文字から始め、英小文字・数字・ハイフンで付ける管理名。例: `my-fox`
- **表示名** — 画面に表示するキャラクター名
- **役割（任意）** — マスコット、案内役、物語の主人公など

作成すると、そのキャラクター専用のフォルダと記録の骨組みが用意されます。

### 2. 基本の見た目を決める

「詞書」で `base` を開き、顔、体格、衣装、色、持ち物など、毎回守りたい特徴を書きます。
サンプルの和紙狐を開くと、書き方の例を確認できます。
自分で編集する場合は、`##` で始まる見出しや、本文を囲む3つのバッククォートを残してください。

> 今決めた内容から、このキャラクターの基本の見た目を詞書にまとめて。

### 3. 作りたい場面を追加する

同じ「詞書」で variant（場面差分）を作ります。
たとえば `three-view` は三面図、`expressions` は表情集、`walking` は歩く場面、といった使い分けです。

> このキャラクターの三面図を作るためのvariantを追加して。

### 4. 生成用プロンプトをまとめる

「調合」で variant を選び、「調合する」を押します。
最初は確認だけなので、まだファイルは増えません。内容に問題がなければ「compile を保存」を押します。

基本の見た目と場面の指定が、ひとつの `prompt.txt` にまとまります。
このプロンプトを、普段使っている画像生成ツールへ渡します。

> 三面図用の内容を調合して、画像生成ツールへ渡せるプロンプトを見せて。

### 5. 結果を記録する

生成結果を見たら「日録」に、良かった点、直したい点、次に試すことを残します。
採用した画像がある場合だけ、「手本」からお手本画像として登録します。候補画像を全部入れる必要はありません。

> この結果の良かった点と直したい点を日録に残して、次の改善案を出して。

次回はその続きから始められます。

```text
設定する → プロンプトをまとめる → 外部ツールで生成する
    ↑                                      ↓
    └──── 評価と次の改善を記録する ────────┘
```

## データはどこに保存される？

キャラクターのデータは、Shitate フォルダ内の `characters/` に保存されます。
1キャラクターにつき1フォルダなので、別のキャラクターと内容が混ざりません。

- プロンプトや記録は Markdown・YAML・JSON という一般的なテキスト形式です
- 調合して保存した結果は、そのキャラクターの `outputs/` に残ります
- お手本画像を登録した場合だけ、そのキャラクターの `references/images/` にコピーされます
- Shitate フォルダ全体を複製すれば、まとめてバックアップできます
- Studio がデータや画像を自動で外部へ送信することはありません

## 次に頼めること

Codex / Claude Codeには、やりたいことを普段の言葉で頼めます。

- 「このキャラの三面図用プロンプトを整えて」
- 「前回の改善メモを読んで、次の案を出して」
- 「この画像からキャラクター設定を書き起こして」
- 「動画制作へ渡せる形にまとめて」

Shitateには作業ルールが同梱されています。
Codex / Claude Codeも過去の記録を読み、前回の続きから作業できる設計です。

## 画像や動画とのつながり

Shitate は生成ツールを固定しません。
Codex / ChatGPT、Claude Code、PixVerse など、そのとき使いたい道具へプロンプトや anchor を渡せます。

画像生成は、利用者が明示的に「生成して」と頼んだときだけ行う運用です。
候補が複数ある場合も、採用した画像だけを正式なお手本として残します。

動画制作へ進む場合は、[Tsugite](integrations/tsugite.md) へキャラクターのスナップショットを渡せます。
資料に基づく人物づくりでは、知識wikiの [makimono](https://github.com/Takamasa045/makimono) とも連携できます。どちらも任意です。

## 安心して試せるための設計

- 1キャラクターを1フォルダに分け、別のキャラと混ぜません
- 既存の生成記録や anchor を、同じ名前で上書きしません
- 変更理由と次の改善を日録に残します
- Studio の書き込み先は、このリポジトリ内に限定します
- 画像生成や外部サービスへの送信は、自動では行いません
- `pnpm distribution:check` で、キャラクター画像や個人端末のパスが配布物へ混ざっていないか確認できます
