# Template: log-entry

`characters/<id>/log.md` に追記する1エントリの雛形。
エントリは **新しい順（DESCENDING）** で追記する（FR-LOG-03）。

---

```markdown
## <YYYY-MM-DD> / base v<N> / <variant>

- **試行**: <何を変えたか。1-3行で具体的に>
- **プロンプト差分**: [<参照先>](<相対パス>)
- **生成物**: [<run-id>](outputs/<run-id>/manifest.json)
- **評価**: <◎|◯|△|✗>
  - 顔の一貫性: <◎|◯|△|✗>
  - 衣装: <◎|◯|△|✗>
  - ポーズ: <◎|◯|△|✗>（具体的な破綻箇所）
  - 背景: <◎|◯|△|✗>
- **次の改善**: <次に試すこと。次回の Claude Code が読んで実行する想定で書く>
```

---

## 評価記号

- **◎** 完璧。これで確定してよい
- **◯** 概ね良い。細部の調整のみ
- **△** 方向は合っているが破綻あり
- **✗** 方向転換が必要

## ルール

- 「次の改善」は **空にしない**。何もなければ「このバリアントは確定。次は別のバリアント」と書く。
- プロンプト差分は `history/base.v<N>.md` か `variants/<name>.md` への相対リンク。
- 生成物リンクは manifest.json への相対パス（画像本体はリポに無いかもしれないが manifest は残る）。
- 作成・改善・安定化の3種類のエントリを最低限揃えると育成ループが成立する。

## 特殊エントリ

### 作成

```markdown
## <YYYY-MM-DD> / base v1 / creation

- **試行**: 新規作成。<出典エンティティ>を参考に初期デザインを設定。
- **プロンプト差分**: [base.v1.md](prompts/history/base.v1.md)
- **生成物**: （未生成）
- **評価**: — （初回）
- **次の改善**: アンカー画像の生成。three-view で同一性の起点を作る。
```

### 安定版認定

```markdown
## <YYYY-MM-DD> / base v<N> / stable-promotion

- **試行**: status を stable に昇格。
- **プロンプト差分**: なし
- **生成物**: 直近3回の生成で顔・衣装・ポーズ全て ◎
- **評価**: ◎ （安定版）
- **次の改善**: 新規バリアント開発に移行。base は凍結。
```
